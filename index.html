1. What is the main function of an operating system?
An operating system manages computer hardware and provides a platform for running application programs, acting as an interface between user and hardware.
2. What does GUI, DOS stand for?
GUI = Graphical User Interface
DOS = Disk Operating System
3. Define pre-emptive and non-pre-emptive scheduling.
Pre-emptive scheduling: The CPU can interrupt a running process (preempt it) to switch to a higher-priority process.

Non-pre-emptive scheduling: Once a process gets the CPU, it runs until it completes or voluntarily yields the CPU.
4. Which memory is directly accessible by the CPU?
Primary memory (RAM) is directly accessible by the CPU.
5. Name any one type of system call.
Process control (e.g., fork or create_process)
OR
File management (e.g., open)
6. What is memory management in an operating system?
Memory management is the OS function that handles allocation, deallocation, and organization of main memory for running processes.
7. What is paging?
Paging is a memory-management scheme where physical memory is divided into fixed-size blocks called frames, and logical memory into same-size pages.
8. What is file management in an operating system?
File management involves creating, deleting, organizing, and accessing files on storage devices, providing file-system services to users.

A process in an OS goes through several states:
  • New: Process is created
  • Ready: Process is waiting in queue for CPU
  • Running: CPU is executing process instructions
  • Waiting: Process is blocked for I/O or event
  • Terminated: Process has finished execution

OS structures define how OS components are organized:

3. What is an Operating System? Explain its main functions.
An Operating System is system software that manages hardware resources and provides services to application programs.

Main functions:
  • Process management: Create, schedule, terminate processes
  • Memory management: Allocate, deallocate memory
  • File system management: Organize, access files
  • Device management: Manage I/O devices
  • Provide user interface: CLI or GUI
4. Explain processor and User modes. Describe how the user interacts with the operating system (interface).
Processor modes:
  • Kernel mode: OS has full access to hardware
  • User mode: User programs run with restricted access

User interacts with OS via:
  • Command-Line Interface (CLI): Text commands
  • Graphical User Interface (GUI): Visual icons, windows
5. Explain the different types of file operations performed by an operating system.
Common file operations:
  • Create: Allocate space and entry in directory
  • Open: Load file attributes into memory
  • Read/Write: Transfer data between file and process memory
  • Close: Release file resources
  • Delete: Remove file entry and free space
6. Explain various types of cyber attacks in operating systems.
Common OS-related cyber attacks:
  • Malware: Malicious software (viruses, worms)
  • Buffer overflow: Overwriting memory to execute code
  • Privilege escalation: Gaining higher access rights
  • Denial of Service (DoS): Overwhelming OS resources
  • Rootkit: Hiding malicious processes

1. Explain different types of Operating Systems with suitable examples.
  • Batch Operating System: Executes batch jobs, no user interaction (e.g., payroll processing)
  • Time-Sharing Operating System: Multiple users share CPU, interactive (e.g., UNIX)
  • Distributed Operating System: Manages distributed resources, networked systems (e.g., cloud computing)
  • Real-Time Operating System: Predictable response time, critical applications (e.g., embedded systems)
  • Network Operating System: Manages network resources, provides services (e.g., Windows Server)
  • Mobile Operating System: Designed for mobile devices, touch interface (e.g., Android, iOS)
2. Explain the First Come First Serve (FCFS) scheduling algorithm. Write its algorithm statement and illustrate it with an example.
FCFS Algorithm: Execute processes in arrival order
Algorithm: 1. Process arrives, add to ready queue 2. CPU executes process until completion 3. Repeat for next process in queue
Example: Processes P1, P2, P3 arrive at 0, 2, 4 ms
P1: 0-5 ms
P2: 5-8 ms
P3: 8-10 ms
  • ‚úÖ Advantages: Simple, easy to implement
  • ‚ùå Disadvantages: Non-preemptive, may lead to poor performance
3. Explain different types of memory management techniques in Operating system.
  • Contiguous Allocation: Each process in contiguous memory block
  • Paging: Divide memory into fixed-size pages
  • Segmentation: Divide memory into variable-size segments
  • Virtual Memory: Use secondary storage to extend RAM

  • ‚úÖ Advantages: Efficient memory use, protection
  • ‚ùå Disadvantages: Complexity, overhead
