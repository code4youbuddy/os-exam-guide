1. Describe the process states in an operating system with the help of a state diagram.
ğŸ“Œ Introduction
A process is a program in execution. During its lifetime, a process transitions through various states as it is created, scheduled, executed, and terminated. Understanding these states is crucial for process management in operating systems.
ğŸ”„ Process States Explained
  • New (Created): The process is being created. The OS allocates resources and initializes the Process Control Block (PCB). The process is not yet ready to execute.
  • Ready: The process has all necessary resources and is waiting in the ready queue for CPU allocation. Multiple processes can be in the ready state simultaneously, managed by the scheduler.
  • Running: The process is currently being executed by the CPU. Instructions are being fetched and executed. Only one process per CPU core can be in running state at any given time.
  • Waiting (Blocked): The process cannot continue execution until some event occurs (e.g., I/O completion, signal from another process). The process is moved to a waiting queue and the CPU is allocated to another process.
  • Terminated (Exit): The process has finished execution. The OS deallocates resources, removes the PCB, and cleans up memory. The process no longer exists in the system.

3. What is an Operating System? Explain its main functions.
ğŸ“Œ Definition
An Operating System (OS) is system software that acts as an intermediary between computer hardware and user applications. It manages hardware resources, provides services to application programs, and creates an environment for users to interact with the computer efficiently and securely.
ğŸ¯ Main Functions of Operating System
1. Process Management
  • Process Creation & Termination: Creates new processes using system calls (fork, exec) and terminates processes when they complete or encounter errors
  • Process Scheduling: Decides which process gets CPU time using scheduling algorithms (FCFS, Round Robin, Priority Scheduling)
  • Process Synchronization: Coordinates processes to avoid race conditions using semaphores, monitors, and locks
  • Inter-Process Communication (IPC): Enables processes to communicate via shared memory, message passing, pipes, and sockets
  • Deadlock Handling: Prevents, avoids, detects, and recovers from deadlock situations
Example: When you open multiple browser tabs, the OS creates separate processes/threads for each tab, schedules CPU time among them, and ensures they don't interfere with each other's memory.
2. Memory Management
  • Memory Allocation: Allocates memory to processes when they need it and deallocates when done
  • Memory Protection: Ensures processes cannot access each other's memory space
  • Virtual Memory: Provides illusion of larger memory using disk space (paging and swapping)
  • Memory Mapping: Maps logical addresses to physical addresses using page tables
  • Cache Management: Manages CPU cache for faster memory access

4. Explain processor and User modes. Describe how the user interacts with the operating system (interface).
ğŸ” Processor Modes (Execution Modes)
Modern processors support at least two modes of operation to protect the system from malicious or erroneous programs. 1. Kernel Mode (Supervisor/System Mode)
  • Privilege Level: Highest privilege level with unrestricted access
  • Access Rights: Can execute all CPU instructions including privileged instructions
  • Memory Access: Can access all memory locations including kernel memory
  • Hardware Control: Direct access to hardware devices and I/O operations
  • Who Runs: Operating system kernel, device drivers, critical system services
  • Purpose: Execute critical OS functions that require direct hardware access
2. User Mode (Unprivileged Mode)
  • Privilege Level: Restricted privilege level
  • Access Rights: Can only execute non-privileged instructions
  • Memory Access: Limited to allocated user memory space only
  • Hardware Control: No direct hardware access; must use system calls
  • Who Runs: User applications, programs, utilities
  • Purpose: Protect system from user program errors or malicious code

2. Explain the First Come First Serve (FCFS) scheduling algorithm. Write its algorithm statement and illustrate it with an example.
ğŸ“Œ Introduction
First Come First Serve (FCFS) is the simplest CPU scheduling algorithm where processes are executed in the order they arrive in the ready queue. It follows a non-preemptive approach, meaning once a process starts execution, it runs to completion.
ğŸ” Algorithm Description
Working Principle:
  • Processes are arranged in a queue based on arrival time
  • CPU is allocated to the first process in the queue
  • Process executes until completion (non-preemptive)
  • Next process in queue gets CPU after current process finishes
  • Similar to a queue at a ticket counter - first person gets served first
Implementation:
  • Uses FIFO (First In First Out) queue data structure
  • Simple to implement using linked list or array
  • No complex calculations required
